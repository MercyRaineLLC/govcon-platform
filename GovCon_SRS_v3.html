1. Product Overview & Mission

The GovCon Advisory Intelligence Platform is a multi-tenant SaaS application purpose-built for government contracting consulting firms. It automates opportunity discovery, applies AI-driven analysis to solicitation documents, and generates actionable bid intelligence — replacing manual research workflows that currently consume dozens of analyst hours per pursuit.

The platform ingests live federal opportunities from SAM.gov, scores them using a probabilistic 8-factor model, and surfaces only the highest-value matches per client. Consultants manage the full pursuit lifecycle — opportunity identification through bid decision, proposal planning, submission tracking, and compliance monitoring — in a single workspace. As of v3.0, the platform adds a BigQuery-powered historical market intelligence layer that surfaces competition profiles, agency buying patterns, and contractor win histories from real federal award data.

Launch Goal: Enable a consulting firm to onboard a client, ingest relevant federal opportunities, receive AI-scored bid recommendations, generate a compliance matrix and bid strategy document, and view historical competitive intelligence — all within 15 minutes of first login, guided by the built-in Onboarding Wizard.

Opportunity Intelligence

Live SAM.gov ingestion, AI scoring, enrichment from USASpending, win probability display with disclaimer, deadline urgency classification, favorites and recently-viewed tracking.

8-factor logistic probability model with Bayesian calibration produces BID_PRIME / BID_SUB / NO_BID recommendations with full factor breakdown per client-opportunity pair.

Branded portal view for end-clients. Shows their active opportunities, bid decisions, submission history, and uploaded documents relevant to their engagements.

4. AI Intelligence Layer

All AI calls route through a central LLM Router that resolves the active provider per firm, checks a Redis cache (7-day TTL) for repeated requests, calls the appropriate provider, logs token usage and cost, and returns the result. Business logic services have no direct knowledge of which AI provider is active.

4.3 Usage Tracking

Every AI call is logged in ApiUsageLog with provider, model, task type, input/output token counts, estimated cost (USD), cache hit flag, and latency. Admins and Consultants view a 30-day usage summary broken down by task and by provider in Settings. Cache hits are logged at $0 cost.

4.4 LocalAI Integration

LocalAI runs as a containerized service alongside the platform. It exposes an OpenAI-compatible REST API with no external key required. If LocalAI fails during a call, the platform automatically falls back to Claude if a key is available. The backend transparently rewrites localhost URLs to the internal Docker hostname so consultants can enter the external URL in Settings regardless of deployment environment.

4.5 Key Resolution Order

For each AI call: (1) read firm's configured provider from DB; (2) use firm's stored API key for that provider; (3) fall back to server environment variable if DB key is null; (4) throw NO_LLM_KEY if no key is available, which triggers a graceful fallback in all calling services.

5. Opportunity Management

5.1 Ingestion & Enrichment

  • Trigger SAM.gov ingest with optional NAICS code, agency, and set-aside filters (up to 100 results per run). Concurrent duplicate jobs are blocked per tenant.
  • Post-ingest, opportunities are automatically queued for probability scoring via BullMQ worker.
  • Enrichment job fetches historical award data from USASpending: incumbent, award history, competitor count, agency small-business and SDVOSB rates.
  • Enrichment fields stored: historicalWinner, competitionCount, incumbentProbability, agencySdvosbRate, agencySmallBizRate, recompeteFlag, historicalAwardCount, historicalAvgAward, isEnriched.

5.2 Probability Scoring — 8 Factors

NAICS Alignment Set-Aside Match Incumbent Probability Competition Density Agency Relationship Document Intel Score Deadline Urgency Penalty Drag

Scores are produced by an 8-factor logistic sigmoid model (weights sum to 1.0, SCALE=6.0, BIAS=−3.0) then Bayesian-calibrated using the firm's historical win/loss record (Beta-binomial, pseudo-count 10). Penalty drag applies an exponential decay: exp(−totalPenalties / 200,000). All displayed probabilities carry an "est." disclaimer badge.

5.3 Opportunity Detail View

  • Score breakdown panel with per-factor contribution visualization

  • Favorite / star to pin to sidebar; recently-viewed list auto-populates
  • Export Synopsis: professional HTML export with agency cleanup, deadline urgency badge, probability gauge, score breakdown bars, AI scope keyword tags, and competitive intelligence grid

6.2 Bid Decisions

Clients access a read-only branded portal showing their opportunity pipeline, bid decisions, and submission history. Portal tokens are scoped exclusively to the authenticated client's clientCompanyId and cannot access any other client or firm data.

7. Analytics & Forecasting

Pipeline Funnel

Ingested → Scored → Decided → Submitted → Won. Real contract values at each stage with conversion rates.

Win Probability Distribution

Histogram of scored opportunities by probability band. Identifies where the pipeline is concentrated.

Revenue Forecast

Monte Carlo simulation (1,000 runs) over active pipeline with lognormal noise (Box-Muller). P10 / P50 / P90 confidence bands plus HHI portfolio diversification index.

Market Intelligence

NAICS sector trend detection via linear regression slope. Agency profiles with small-business award rates and competitive landscape summary.

Submission Velocity

EMA-smoothed (alpha = 2/(span+1)) trend of submissions over time. Late submission probability modeled via Beta distribution.

Risk Radar

Flags deadline risk, compliance blocks, high late-submission probability (Beta distribution mean), and unpaid penalties requiring immediate attention.

Smart Match Engine

Automatically surfaces top client-opportunity pairings ranked by combined probability score with NAICS pre-filter. Bounded concurrency (5 parallel) for large portfolios.

Penalty Trend

Time-series of financial penalties applied across the portfolio. EMA-smoothed. Identifies compliance risk patterns per client over time.

8. BigQuery Market Analytics New in v3.0

The platform maintains a Google BigQuery analytics store (govcon_analytics dataset, project gov-con-491005) that ingests federal award records from USAspending and makes them available for historical market intelligence queries. This layer is separate from the PostgreSQL operational database and is optimized for aggregate analytics.

8.1 Data Pipeline

  • Source: USAspending.gov public API — award transactions filtered by NAICS code and optional agency
  • Destination: BigQuery table award_history with fields: recipient name, award amount, action date, set-aside type, NAICS code, agency name, number of offers received
  • Ingestion: paginated (up to maxPages × 100 records per call), 500-row insert batches, polite 1-second delay between NAICS codes in bulk mode
  • Pre-seeded: 2,000 award records across 20 federal NAICS codes covering IT, Engineering, Consulting, R&D, Facilities, Defense, Healthcare, Staffing, Construction, and Manufacturing
  • Authentication: Google Application Default Credentials (ADC) via mounted credentials file

8.4 Analytics Engine — Key Metrics Computed

  • HHI (Herfindahl-Hirschman Index): sum of squared market share percentages per winner. HHI < 1,500 = competitive; 1,500–2,500 = moderately concentrated; > 2,500 = concentrated.
  • Top Winners: ranked by total awarded value with market share percentage
  • Set-Aside Breakdown: percentage of awards by set-aside type (SDVOSB, WOSB, SB, unrestricted)
  • Yearly Trend: award volume by fiscal year for trajectory analysis
  • Agency Rates: percentage of awards to small businesses, SDVOSB, WOSB, HUBZone per agency
  • Average Offers Received: market signal for competition density — lower offers = less competitive field

8.5 Frontend Integration

The Analytics page includes a "Deep Market Intelligence" section that shows BigQuery connection status (green/gray badge + row count), and when data is available: 4 top-line metric cards (total market volume, average contract size, unique competitor count, tracked NAICS codes), a NAICS market heatmap using HHI bar visualization, and a top contracting agencies table. If no data is present, a prompt guides the admin to trigger ingestion.

9. Compliance & Audit

  • Compliance Logs: Every status change on opportunities, submissions, and documents generates an immutable audit log entry with entity type, from/to status, triggering user, and timestamp.
  • Compliance Matrix: AI-generated requirement checklist per opportunity. Each row tracks requirement text, FAR reference, mandatory flag, status (NOT_STARTED / IN_PROGRESS / COMPLETE / WAIVED / NON_COMPLIANT), assigned proposal section, and notes.
  • Document Requirements: Per-client checklists for certifications, past performance packages, and financial documents with due dates and compliance status.
  • Compliance Report Export: Date-range filtered export of all audit events for a firm.
  • Compliance State Machine: Enforces legal status transitions on submission records. Invalid transitions are rejected and logged.

11.4 Error Handling

  • ErrorBoundary: Global class component wrapping the entire app. Catches any React render error, displays a branded recovery screen, and provides a "Return to Dashboard" button that resets state and navigates home.
  • 404 Route: Catch-all path="*" renders a styled Not Found page with links back to Dashboard and Opportunities.
  • API Errors: All queries surface errors via TanStack Query's isError state; UI shows ErrorBanner component with the error message.
  • AI Failures: All three AI task types (document analysis, compliance matrix, bid guidance) have explicit fallback paths — they never throw unhandled exceptions to the user.

11.5 Onboarding Wizard

A 5-step modal overlay shown once per browser session on first Dashboard load (controlled via localStorage['govcon_onboarded']). Steps:

  1. Welcome — Platform overview and mission statement
  2. API Keys — Guide to Settings → Add SAM.gov and AI provider keys (with direct navigation link)
  3. Sync Opportunities — How to trigger SAM.gov ingest (with link to Opportunities page)
  4. Add Clients — Client onboarding and portal setup (with link to Clients page)
  5. Run AI Analysis — Compliance matrix and Analytics overview (with link to Analytics page)

Each step includes a navigation shortcut button. The wizard can be dismissed at any step ("Skip setup guide"). Progress is indicated by a step progress bar at the top.

12. Non-Functional Requirements & Beta Readiness

12.1 Security & Multi-Tenancy

  • Every database query scoped by consultingFirmId extracted from JWT — no cross-tenant data leakage is architecturally possible
  • Client portal tokens scoped exclusively to their clientCompanyId
  • Role-based access control enforced at the route middleware level (requireRole() middleware)
  • API keys never returned in list endpoints — only masked previews exposed
  • BigQuery credentials mounted read-only (:ro) into the backend container
  • Rate limiting: 500 requests per 15-minute window per IP

12.2 Reliability & Resilience

  • All AI provider failures return graceful fallbacks — no application crashes or unhandled promise rejections
  • LocalAI failures fall back to Claude automatically if a key is available
  • Background jobs (scoring, enrichment, ingestion) run in isolated BullMQ workers with error capture
  • Uploaded files persist in a named Docker volume independent of container lifecycle
  • Redis caching (7-day TTL) for compliance matrix and bid guidance reduces redundant AI cost
  • BigQuery operations are non-blocking for the main application — BQ failures surface as null data states, not crashes
